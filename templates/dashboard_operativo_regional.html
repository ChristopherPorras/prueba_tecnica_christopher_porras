Tendencia semanal

Comparación de 2024 y 2025 con mensaje flotante al pasar el cursor.

Explicación técnica y consulta usada

La gráfica agrupa por semana y separa por año. Cada transacción completada suma; cada devolución resta. El control superior permite alternar entre ventas netas y transacciones.

SELECT
  DATE(t.transaction_date, '-' || ((CAST(strftime('%w', t.transaction_date) AS INTEGER) + 6) % 7) || ' days') AS semana,
  strftime('%Y', t.transaction_date) AS año,
  SUM(CASE WHEN t.status = 'RETURNED' THEN -t.total_amount ELSE t.total_amount END) AS ventas_netas,
  COUNT(DISTINCT t.transaction_id) AS transacciones
FROM transactions t
JOIN stores s ON s.store_id = t.store_id
WHERE DATE(t.transaction_date) BETWEEN DATE('__DEFAULT_START__') AND DATE('__DEFAULT_END__')
GROUP BY semana, año
ORDER BY semana, año;

Ranking de tiendas

Haz clic en una tienda para ver su detalle operativo.

0 tiendas Todas
Explicación técnica y consulta usada

El ranking agrega ventas por tienda, calcula ticket promedio y ventas netas por metro cuadrado. Luego compara cada tienda contra el percentil 25 de su propio formato para marcar bajo rendimiento.

WITH ventas_tienda AS (
  SELECT
    s.store_id,
    s.store_name,
    s.format,
    s.size_sqm,
    SUM(CASE WHEN t.status = 'RETURNED' THEN -t.total_amount ELSE t.total_amount END) AS ventas_netas,
    COUNT(DISTINCT t.transaction_id) AS transacciones
  FROM transactions t
  JOIN stores s ON s.store_id = t.store_id
  WHERE DATE(t.transaction_date) BETWEEN DATE('__DEFAULT_START__') AND DATE('__DEFAULT_END__')
  GROUP BY s.store_id, s.store_name, s.format, s.size_sqm
),
scored AS (
  SELECT
    *,
    ventas_netas / NULLIF(size_sqm, 0) AS ventas_netas_por_metro_cuadrado,
    ventas_netas / NULLIF(transacciones, 0) AS ticket_promedio
  FROM ventas_tienda
)
SELECT *
FROM scored
ORDER BY ventas_netas_por_metro_cuadrado DESC;

Retención de clientes de lealtad por cohorte

Color más intenso significa mayor porcentaje de clientes que regresan.

Explicación técnica y consulta usada

Una cohorte es el mes de primera compra del cliente identificado. La retención del mes 1, 2, 3 o 6 es el porcentaje de clientes de esa cohorte que vuelve a comprar en ese mes relativo.

WITH ventas_lealtad AS (
  SELECT
    customer_id,
    DATE(transaction_date, 'start of month') AS mes_compra,
    total_amount
  FROM transactions
  WHERE loyalty_card = 1 AND customer_id IS NOT NULL AND status = 'COMPLETED'
),
primera_compra AS (
  SELECT customer_id, MIN(mes_compra) AS mes_cohorte
  FROM ventas_lealtad
  GROUP BY customer_id
)
SELECT mes_cohorte, COUNT(DISTINCT customer_id) AS clientes
FROM primera_compra
GROUP BY mes_cohorte
ORDER BY mes_cohorte;

Participación por formato

Haz clic en un formato para filtrar el dashboard.

Explicación técnica y consulta usada

Este componente resume ventas netas por formato sobre los filtros activos. La barra representa participación relativa frente al formato con mayor venta.

SELECT
  s.format AS formato,
  SUM(CASE WHEN t.status = 'RETURNED' THEN -t.total_amount ELSE t.total_amount END) AS ventas_netas,
  COUNT(DISTINCT t.transaction_id) AS transacciones
FROM transactions t
JOIN stores s ON s.store_id = t.store_id
WHERE DATE(t.transaction_date) BETWEEN DATE('__DEFAULT_START__') AND DATE('__DEFAULT_END__')
GROUP BY s.format
ORDER BY ventas_netas DESC;

Quiebres activos de venta

Priorizados por ventas estimadas pérdidas.

Explicación técnica y consulta usada

Un posible quiebre aparece cuando un producto que históricamente se vendía en una tienda deja de venderse por tres o más días. La pérdida estimada usa el promedio diario de ventas de los 14 días previos multiplicado por la duración del período sin venta.

WITH ventas_diarias AS (
  SELECT
    t.store_id,
    ti.item_id,
    DATE(t.transaction_date) AS fecha,
    SUM(ti.quantity * ti.unit_price) AS ventas
  FROM transaction_items ti
  JOIN transactions t ON t.transaction_id = ti.transaction_id
  WHERE t.status = 'COMPLETED'
  GROUP BY t.store_id, ti.item_id, DATE(t.transaction_date)
)
SELECT
  store_id,
  item_id,
  MAX(fecha) AS ultima_fecha_con_venta,
  CAST(JULIANDAY('2025-06-30') - JULIANDAY(MAX(fecha)) AS INTEGER) AS días_sin_venta
FROM ventas_diarias
GROUP BY store_id, item_id
HAVING JULIANDAY('2025-06-30') - JULIANDAY(MAX(fecha)) >= 3;
